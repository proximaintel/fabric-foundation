# Contributing

We welcome contributions to improve this foundation.

## How to Contribute

1. **Open an issue** — describe the improvement or bug
2. **Fork the repo** — make changes on a feature branch
3. **Run `az bicep build`** — ensure Bicep compiles cleanly
4. **Submit a pull request** — reference the issue

## Guidelines

- Bicep must pass `az bicep build` without errors
- Terraform must pass `terraform validate`
- Follow Microsoft naming conventions for Fabric resources
- Update documentation when changing module interfaces
- All contributions must be compatible with the Apache 2.0 license

## Questions?

Open an issue or reach out at [proximaintel.com/contact](https://proximaintel.com/contact).
